# Paper Agent Documentation

This directory contains versioned documentation for the Paper Agent project.

## What is Paper Agent?

Paper Agent is a CLI-first paper intelligence tool designed for AI researchers to collect papers from arXiv and conferences, use LLM for semantic filtering, generate reports, and integrate with automation tools like Cursor and Codex.

## Documentation Versions

- **[v01](./v01/)** - Initial version defining requirements, features, specifications, design, and MVP scope

## Reading Order

For each version, follow this recommended reading order:

1. **Requirement** - Understand the problem, users, and goals
2. **Spec** - Learn the functional specifications and feature scope
3. **Design** - Review the technical architecture and implementation approach
4. **Feature** - Review the feature view derived from the design and system capabilities
5. **MVP** - See what's included in the minimum viable product

## Versioning Convention

- Each version is stored in a `vXX/` directory (e.g., `v01/`, `v02/`)
- Version numbers are zero-padded two-digit integers
- Each version contains: `requirement.md`, `feature.md`, `spec.md`, `design.md`, `mvp.md`
- Major changes or pivots warrant a new version
