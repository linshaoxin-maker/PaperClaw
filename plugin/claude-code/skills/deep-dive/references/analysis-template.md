# Paper Analysis Template (Enhanced)

## 使用说明

深度分析笔记的标准输出格式。文件通过 `paper_note_add` 保存到 `.paper-agent/notes/{paper_id}.md`。

**关键规则**：
1. 必须先问用户关注什么角度，不要千篇一律输出所有章节
2. "与我研究的关联" 必须参考 `paper_profile` 的研究方向
3. 技术术语保留英文，解释用中文

## 模板

```markdown
# {论文标题} — 深度分析

> 分析角度: {用户选择的角度，如"方法创新 + 与我研究的关联"}
> 分析日期: {YYYY-MM-DD}
> 论文状态: {reading_status}

## 核心信息

| 字段 | 值 |
|------|---|
| ID | {paper_id} |
| arXiv / DOI | {source_paper_id} |
| 作者 | {authors} |
| 机构 | {affiliations, 从 metadata 提取} |
| 发布时间 | {published_at} |
| 来源 | {source_name} |
| 链接 | {url} |
| 评分 | {relevance_score}/10 |

## 摘要翻译

{将 abstract 翻译为中文，保留技术术语原文}

## 要点提炼

1. {Key point 1 — 这篇论文最重要的贡献}
2. {Key point 2}
3. {Key point 3}

---
以下章节根据用户选择的分析角度展开，未选择的角度省略。
---

## 方法创新点 {如果用户选了 a}

### 核心思想
{一段话概括核心创新}

### 方法框架
{整体架构描述，如果有多个模块则逐个说明}

### 与前人方法的区别
| 维度 | 本文方法 | 前人方法 | 改进点 |
|------|---------|---------|--------|
| {维度1} | ... | ... | ... |

## 实验设计 {如果用户选了 b}

### 实验设置
- Benchmark: {使用的数据集/benchmark}
- Baseline: {对比的方法}
- 指标: {评估指标}

### 主要结果
| Benchmark | 指标 | 本文 | Best Baseline | 改进 |
|-----------|------|------|--------------|------|
| ... | ... | ... | ... | ... |

### 消融实验
{消融实验的关键发现}

## 与我研究的关联 {如果用户选了 c}

> 以下分析基于你的研究方向: {从 paper_profile 获取的 topics 和 keywords}

### 交叉点
{这篇论文跟用户研究方向的交叉在哪里}

### 可借鉴的方法
{哪些方法/技巧可以用到用户的研究中}

### 启发
{对用户当前工作的启发和建议}

## 局限与改进空间 {如果用户选了 d}

### 局限性
1. {limitation 1}
2. {limitation 2}

### 改进方向
1. {可能的改进方向 1}
2. {可能的改进方向 2}

### 适用场景
{这个方法最适合什么场景，不适合什么场景}

## 研究价值评估

| 维度 | 评分 | 说明 |
|------|------|------|
| 创新性 | {X}/10 | {一句话} |
| 实用性 | {X}/10 | {一句话} |

## 行动建议

- [ ] {建议 1，如"复现 Section 3 的方法"}
- [ ] {建议 2，如"追踪引用链看后续改进"}
- [ ] {建议 3，如"跟 XX 论文做对比"}
```
